NiftyCoin - an art and media enabling blockchain and crypto solution

NiftyCoin and the NiftyBlockchain is built upon the LiteCoin design, which again is a fork of BitCoin. NiftyCoin has been designed to enable persistant media and art content, so that they will forever be traceable, from birth, and whenever the media or art content changes hands.

Therefore the storage capacity for each transaction has been increased beyond what BitCoin and LiteCoin can offer,  and a best practice protocol has been designed to store either a reference or the actual content within the blockchain.

A whitepaper is in the works to describe this in detail.

 

 

Protocol Information

Here are some highlevel protocol information (in the request for change NCE-02 )

Stored meta-data:

  • Main Artist / Designer / Creator (can be multiple)
  • If more than one Artist / Designer / Creator - the redistribution share is denoted (will be used to share the proceeds every time a  media or art piece is sold). 
  • Title of Art Piece / Design / Song / Sample
  • For art / designs - Number of identical pieces (e.g. 1000)
  • For art / design - What number of the art is this  (e.g. numer 54 - out of the 1000 above)
  • For music and sound samples - Reference to Album and index
  • Agreed redistribution percentage to blockchain fascilitator 
  • Unique File Identity - from a proposed International Art and Media Database

 

Technical details:

Field Size Description
Identifier 4 bytes NFTY or 0x4e 0x46 0x54 0x59
Quantity multiple 4 bytes Multiply the asset quantity in display units by this number in order to calculate raw units. Display units are used for NiftyChain API calls, whereas raw units are used within transaction OP_DROPs. Stored as an unsigned 32-bit integer in small-endian order, e.g.: 0x64 0x00 0x00 0x00 = 100 for an asset where 1 raw unit = 0.01 display units.
Asset name Variable Null-delimited name of asset, e.g. 0x61 0x73 0x73 0x65 0x74 0x31 0x00 for asset1.
Repeat the below for each additional asset property
Property key Variable If the first byte of the key is 0x00, it denotes a property with special meaning to NiftyChain, and the second byte gives the property type. For now, the only possible key is 0x00 0x02, where a property value of 0x00 or 0x01 denotes whether the asset is open for reissuance. If the first byte of the property key is not 0x00, it contains the null-delimited name of a user-defined custom field, e.g. 0x75 0x72 0x6c 0x00 for url.
Length 1-9 bytes Bitcoin-style variable-length integer indicating the length of the property value in bytes.
Value Variable The property’s value as raw binary.
